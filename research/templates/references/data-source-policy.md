# 数据来源与审计线索规范

> 适用：所有遵循 `templates/company-analysis.md` 的研究报告。
> 版本：0.2（与主模板同步）

研究报告的可信度取决于"每一个关键数字都能被反查"。本文件规定来源优先级、必标项与口径要求。任何 agent 在产出报告前必须读取本文件。

---

## 1. 来源优先级

写入数字 / 引用观点 时，优先级**自上而下**。当多源冲突，以高优先级为准（并在文中说明冲突）。

| 等级 | 类型 | 示例 |
|------|------|------|
| **L1 一手权威** | 公司公告、年报/季报、招股书、交易所披露、公司 IR、监管批复 | SMIC 2024 年报、港交所披露易、SEC 10-K/10-Q |
| **L2 机构数据库** | 持牌金融数据库 | Wind / Bloomberg / Choice / Capital IQ / FactSet |
| **L3 行业研报 / 智库** | 卖方深度报告、行业协会、咨询机构 | Gartner、IDC、SEMI、券商深度 |
| **L4 媒体 / 公开报道** | 主流财经媒体 | 路透、FT、彭博新闻、财新、第一财经 |
| **L5 非结构化网络资源** | 维基、博客、社交媒体、web search 摘要 | Wikipedia、个人博客、推特帖 |

**硬规则**：
- **关键财务、估值、市场份额数字**必须来自 **L1 或 L2**，不能以 L4/L5 作为主来源
- L3 可作为对比/校验，但结论性引用必须能在 L1/L2 中找到对应数据
- L5（含 web search 摘要）**仅可用于事件、新闻、定性背景补充**，不能用作硬数字来源
- 中英文媒体冲突时，看公司主披露语言；港股以港交所为准，A 股以上交所/深交所为准，美股以 SEC 为准

---

## 2. 按数据类别的来源矩阵

不同数据类别的"主来源"要求不同。生成报告时先按本表确定来源级别，再套用第 3 节审计线索规则。

| 数据类别 | 首选来源 | 可接受备用 | 禁止作为主来源 |
|----------|----------|------------|----------------|
| 公司财务数据 | 年报、季报、招股书、交易所披露、SEC/HKEX/上交所/深交所文件 | 持牌数据库、公司 IR 汇总页 | 新闻稿、媒体报道、搜索摘要 |
| 市场价格 | 交易所/官方行情、可靠行情库 | yfinance、akshare 等工具源（需标注交易日） | 无来源网页截图、论坛报价 |
| 估值乘数 | 按同一交易日股价与同一口径财务数据自行计算 | 持牌数据库、公司/券商材料（需标明口径） | "网上平均 PE"、无样本说明的行业均值 |
| 市场份额 / 行业规模 | 行业协会、政府统计、公司披露 | 咨询机构、券商研报（需标注日期和样本范围） | 自媒体、无出处 blog、搜索摘要 |
| 新闻 / 事件 | 公司公告、交易所公告、监管文件 | Reuters、Bloomberg、财新、FT 等主流媒体 | 社交媒体传闻、营销号 |
| 舆情 / 市场情绪 | 明确定义的方法论指标（如 VIX、Fear & Greed） | 社交媒体统计（必须写明采样方法） | 把零散评论当成结论 |

**冲突处理**：
- 高等级来源覆盖低等级来源。例如 L1 年报与 L4 媒体数字冲突，以年报为准。
- 同等级来源冲突时，在正文说明差异、选择理由和影响。
- 如果只能找到低等级来源，必须在 `## Meta → 数据缺口` 中说明，并将 frontmatter `confidence` 调低到 `medium` 或 `low`。

## 3. 审计线索三必标

**任何关键数字旁边，必须能让读者一眼看出三件事**：

1. **`as_of`** —— 数据截止日（不是引用日期，是数据本身的对应时点）
2. **来源** —— 机构名 + 文档名 / 数据库 + 字段
3. **口径** —— 币种、财年（FY/CY）、TTM/LTM、合并/母公司、自然年/会计年

### 表达方式

最少形式：在表格题注或行尾注明。例：

```
| 指标 | FY2024 |
|------|--------|
| 营业收入 | $80.30 亿 |
| 净利润   | $4.93 亿 |

> 来源：SMIC 2024 年报（2025-03 发布），口径：合并报表、美元、自然年、未审核 → 已审核
> as_of：2024-12-31
```

行内形式：`营收 803 亿美元（FY2024，来源：年报，as_of 2024-12-31）`。

推荐短格式：`营收 80.3 亿美元（FY2024；as_of 2024-12-31；来源：2024 年报；口径：合并报表）`。

### 估值结论必须可反查

报告里出现的 PE/PS/PB/EV-EBITDA/DCF 数字，都必须在文档内（或 frontmatter `sources`）找到对应的输入：
- 股价的 `as_of`
- EPS / 营收 / EBITDA 的来源与口径
- 同业可比公司选取标准
- DCF 的核心假设（WACC、增长率、终值倍数）

**反例**（不允许）：
> 当前 PE 188 倍，估值偏高。

**正例**：
> 当前 PE 188 倍（A 股 688981，股价 ¥118.73 @ 2026-04-30，EPS TTM ¥0.63 来自 Q4 2025 财报口径）。同业 TSMC PE 约 25 倍，三星 12 倍——SMIC 显著溢价主要来自国家战略叙事。

## 4. 估算与假设标注

凡是**不是直接来自来源、由 agent 推算或假设**的数字，必须在数字后**显式标注**：

- `(估算)` —— 基于公开数据推算
- `(假设：xx)` —— 引入了某个假设条件
- `(口径调整：xx)` —— 对原始数据做了口径转换

例：

> FY2025 营收 ~$93.3 亿（估算，基于 Q1-Q4 季度数据加总）
> 净利率 ~7.3%（估算，假设：Q4 一次性损益不再发生）

推荐短格式：

> 2026E 营收 92 亿美元（估算；假设：全年增速 14%；基准：FY2025 实际）

**不允许**直接写一个无标注的"看起来像精确数字"的数（如 `$93.3 亿` 不带标注），这会误导读者以为是官方数据。

---

## 5. 口径冲突与时点统一

- **同一份报告内**，同一指标必须使用同一口径。如 FY2024 营收一处写美元、另一处写人民币，必须在表格题注说明
- **跨年度对比**必须使用同一口径（如不能 2023 用合并报表、2024 用母公司报表）
- **多市场对比**（如 SMIC A/H 双股）：股价取同一交易日的收盘价，估值倍数取同一口径的 EPS

---

## 6. Web Search 使用边界

允许：
- 查找事件、新闻、产品发布、监管动态
- 验证公司近况、寻找官方披露入口
- 收集定性观点作为参考

禁止：
- 把 search 摘要直接当成硬数字来源（"网上说营收 100 亿"是不能用的）
- 引用没有具体来源链接的二手汇总
- 引用与主披露语言/监管口径冲突的非权威数据

如果 web search 是唯一渠道，必须在 `## Meta` 的"数据缺口"中明确说明，并在 frontmatter `confidence` 字段降级为 `medium` 或 `low`。
