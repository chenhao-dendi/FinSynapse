# 估值方法规范

> 适用：所有遵循 `templates/company-analysis.md` 的研究报告，对应主模板 §6 估值章节。
> 版本：0.1（随主 spec v0.2 引入）

本规范定义三类估值方法的最小撰写要求。**目的不是教方法本身**（金融教科书已经讲过），而是：
1. 明确哪些输入必须显式标注（呼应 `data-source-policy.md`）
2. 防止常见错误（敏感性被单一假设支配、循环引用同业、无视周期）
3. 让所有报告的估值章节可对比、可审计、可重算

---

## 1. 三类方法的适用与组合

| 方法 | 适用场景 | 弱点 |
|------|---------|------|
| **相对估值（Comps / Multiples）** | 任何有同业可比公司的标的 | 同业选错则结论错；周期顶/底易误判 |
| **历史分位（Historical Quantile）** | 有 ≥3 年 PE/PB/PS 历史的成熟公司 | 周期变化、商业模式切换会让历史失效 |
| **DCF（绝对估值）** | 现金流可预测、终值假设可承受质疑的公司 | 假设敏感度极高；新技术 / 周期股不适用 |

**主 spec §6 的最低要求**：
- §6.1 必须用 **相对估值**（PE/PB/PS/EV-EBITDA + 历史分位 + 同业对比）
- §6.2 多情景估值至少含 Bull/Base/Bear，**情景假设必须有质的差异**（详见 quality-checklist B3）
- DCF 是**可选模块**——若做，必须按本文件 §3 的格式写

## 2. 相对估值（Comps）

### 2.1 必备输入

| 输入 | 必标 | 说明 |
|------|:----:|------|
| 股价 | ✅ | 含 `as_of` 与币种 |
| 流通股本 / 市值 | ✅ | 含口径（A 股 / 全流通） |
| EPS（TTM 或 FY+1E） | ✅ | TTM = 滚动 12 个月；FY+1E = 一致预期 |
| 营收（TTM 或 FY+1E） | ✅ | 同上 |
| EBITDA / 净利润 | ✅ | 同上；EBITDA 需说明加回项 |
| 净负债 / 现金 | EV 倍数必标 | 计算 EV = 市值 + 净负债 |

### 2.2 同业可比公司选择标准

**至少满足 2 项**：
- 主营业务相同子赛道（如同为晶圆代工，不能拿 Foundry 比 Fabless）
- 营收规模在 0.3x - 3x 范围（避免与 5x+ 的巨头硬比）
- 主要市场地域相近（中国大陆 vs 全球）
- 同处可比的周期阶段
- 上市时间 ≥3 年（避免 IPO 后短期估值噪音）

**禁忌**：
- ❌ 用全球龙头（如 TSMC、NVIDIA）作为唯一可比 → 必然显得标的"便宜"
- ❌ 只挑选估值最高的同业 → 高估
- ❌ 跨行业比较（如把 SMIC 和 SaaS 公司比 PE）

### 2.3 倍数选择

| 标的特征 | 推荐倍数 |
|---------|---------|
| 稳定盈利、低波动 | PE / PB |
| 高 Capex / 高折旧（如代工厂、运营商） | EV/EBITDA |
| 收入快速增长但未盈利 | PS / EV/Sales |
| 资产驱动（银行、保险、地产） | PB / P/EV / 内含价值 |
| 周期股 | 周期均值 PE / 历史分位 |
| 早期 / 亏损（如 SaaS、生物科技） | PS / EV/ARR / Rule of 40 |

⚠️ **绝对禁忌**：单用 PE 给周期股估值（盈利低点 PE 看起来"高"，盈利高点 PE 看起来"低"，方向反了）。

### 2.4 撰写格式（写在主模板 §6.1）

```markdown
| 公司 | PE (TTM) | PB | PS | EV/EBITDA | 备注 |
|------|---------|-----|-----|-----------|------|
| <标的> | ... | ... | ... | ... | <as_of, 币种, 口径> |
| 同业 1 | ... | ... | ... | ... | <可比说明> |
| 同业 2 | ... | ... | ... | ... | ... |
| 同业平均 | ... | ... | ... | ... | 算术平均 / 中位数（标注） |
| 历史 5Y 平均 | ... | ... | ... | ... | <数据来源> |
| 历史分位 | ... | ... | ... | ... | 当前值在历史 5Y 序列的分位 |
```

## 3. DCF（可选模块）

### 3.1 何时**不要**做 DCF

- 商业模式刚刚切换（如转型 SaaS、并购重大资产）
- 主营业务受单一政策 / 监管事件主导（如新药审批、出口管制）
- 早期公司、新兴技术（特斯拉早期、AI 模型公司）
- 强周期股的周期顶部（DCF 会把高峰盈利外推成"永远"）

这些场景下，DCF 看起来"科学"，但假设偏差会被复利放大，结论比相对估值更不可靠。

### 3.2 必备输入与显式标注

| 输入 | 必须标注 |
|------|---------|
| 预测期长度 | 5-10 年 |
| 营收增长率 | 每年单独列；不允许"平均增长 X%" 一句话带过 |
| 利润率假设 | 毛利率 / 经营利润率 / 净利率 趋势 |
| Capex 与折旧 | 比例或绝对值；明确周期阶段 |
| 营运资本变动 | 应收 / 存货 / 应付 假设 |
| 终值（Terminal Value） | 永续增长率（≤2-3%）或退出倍数（与同业对比） |
| **WACC（贴现率）** | Rf + Beta×ERP + 国家风险溢价；分项列出 |
| 税率 | 实际税率 vs 法定税率，差异说明 |

### 3.3 敏感性分析（强制）

至少做**双变量敏感性表**：

```markdown
|              | WACC -1% | WACC | WACC +1% |
|--------------|---------|------|---------|
| 永续增长 -1% | ...     | ...  | ...     |
| 永续增长     | ...     | **基准** | ... |
| 永续增长 +1% | ...     | ...  | ...     |
```

⚠️ **若 WACC ±1% 或永续增长 ±1% 导致估值变动 >50%**，必须在文中显式声明"DCF 结论高度依赖单一假设，仅作参考"——这与 quality-checklist B4 呼应。

### 3.4 与相对估值的交叉检验

DCF 的 implied PE / EV/EBITDA 应与同业可比有合理对应。如果 DCF 算出来比同业 1.5x 还贵，要么 DCF 假设过乐观，要么同业可比有结构性差异——必须解释。

## 4. 历史分位

### 4.1 用法

写在主模板 §6.1 当前估值表的 "历史分位" 列：
- 计算口径：当前值在过去 N 年（推荐 5Y，最少 3Y）的同一倍数序列中的百分位
- 例：PE 当前 188 倍，过去 5Y 在 60-200 倍区间，当前位于 92% 分位 → 接近历史最高

### 4.2 何时历史分位无效

- 商业模式发生质变（如苹果从硬件转服务）
- 上市时间 <3 年
- 周期阶段切换（如从下行周期转向 AI 叙事驱动）

这些情况下需在文中说明"历史分位仅供参考，不作为估值锚定"。

## 5. 多情景估值（主模板 §6.2 强制要求）

### 5.1 三档情景的最低要求

| 情景 | 必须不同的维度 |
|------|--------------|
| **Bull** | 至少 1 个核心增长假设激进 + 至少 1 个利润率假设激进 |
| **Base** | 当前一致预期 / 公司指引 / 中性周期 |
| **Bear** | 至少 1 个核心增长假设保守 + 至少 1 个风险事件落地 |

⚠️ 三档不能只是数字微调（+10% / 0 / -10%）。每档必须写明**质的不同假设**——这是 quality-checklist B3 强制项。

### 5.2 隐含 PE 反算

每个情景下用预测净利润 / 当前股价 算出 隐含 PE，与同业 / 历史分位对比：
- **Bull 隐含 PE < 同业平均** → 当前估值已合理（甚至低估）
- **Base 隐含 PE ≈ 同业平均** → 估值合理
- **Bear 隐含 PE >> 同业平均** → 当前估值无支撑，存在大幅下行风险

## 6. 自检（写在主模板 §6 时核对）

- [ ] §6.1 倍数选择与标的特征匹配（周期股不要单用 PE）
- [ ] 同业可比至少 3 家，每家都标了"可比说明"
- [ ] 关键估值数字都有 `as_of` 和来源（呼应 data-source-policy）
- [ ] 多情景假设有质的差异（B3）
- [ ] 若做 DCF：双变量敏感性表已列；单一假设依赖已声明（B4）
- [ ] 估值结论与 §3 护城河 / §5 风险章节的判断方向一致（不能业务"中性"但估值给"积极"）

## 7. 反例与正例

### 反例
- ❌ "PE 25 倍合理" — 没说基准（同业？历史？DCF？）
- ❌ Bull/Base/Bear 三档营收差 ±10%、利润率差 ±2pp — 不是质的差异
- ❌ 用 TSMC 的 PE 给 SMIC 估值 — 全球龙头不可比
- ❌ DCF 用 5% 永续增长 — 远超长期 GDP 增速，几乎必然高估
- ❌ 周期股顶部用 PE — 盈利高峰 PE 低，但马上要下行

### 正例
- ✅ "当前 PE 188（@2026-04-30），同业 12-25，历史 5Y 80-200，分位 70%"
- ✅ Bull 假设：AI 需求超预期 + 新厂全部满产 + N+3 良率突破；Base：成熟稳定；Bear：制裁升级 + 建设受阻
- ✅ DCF 配双变量敏感性 + "若 WACC ±1% 估值变动 >50%，结论谨慎参考"
- ✅ 周期股用周期均值 PE 或 EV/EBITDA + 同业横向对比
