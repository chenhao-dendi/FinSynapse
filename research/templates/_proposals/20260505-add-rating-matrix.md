## 问题描述

- 现有模板第 7 章只有一个**单一标签**（积极 / 中性 / 谨慎 / 回避），但不同维度的强弱被压缩成一个标签会丢失信息
- 旧报告（迁移参考）使用了 5 维 ★★★★★ 评分（战略地位 / 技术实力 / 财务健康 / 成长潜力 / 抗风险），保留了维度颗粒度
- 在 SMIC 这种"战略 ★★★★★ + 财务 ★★★☆☆"差异极大的标的上，单一"中性"标签无法表达内部张力

## 建议改法

在第 7 章新增 **7.0 维度评级矩阵**（或放在 7.1 之前）：

```markdown
### 7.0 维度评级矩阵

| 维度 | 评分（5 档） | 一句话理由 |
|------|:----------:|----------|
| 业务地位 | ★★★★★ / ★★★★ / ★★★ / ★★ / ★ | |
| 财务健康 | ... | |
| 护城河 / 竞争 | ... | |
| 治理与资本配置 | ... | |
| 成长驱动 | ... | |
| 风险抵御 | ... | |
```

5 档定义：
- ★★★★★ 极强 / 极优 / 极清晰
- ★★★★ 较强 / 较优 / 较清晰
- ★★★ 中性 / 一般
- ★★ 较弱 / 存在明显问题
- ★ 极弱 / 重大瑕疵

7.1 单一标签保留，但在前面加 7.0 矩阵作为分维度细化。

## 优先级

**高**——多份报告都会受益，且改动小（不影响骨架，仅在第 7 章前置加一节）
